# 项目概述与架构

## 项目简介

一个基于Vue 3 + TypeScript + Axios构建的前端项目，提供了完善的API请求封装、状态管理和类型定义系统。项目采用模块化设计，便于维护和扩展。

## 技术栈

- **前端框架**: Vue 3
- **TypeScript**: 提供类型安全
- **状态管理**: Pinia
- **HTTP请求**: Axios (封装版)
- **进度条**: NProgress
- **环境管理**: Vite环境变量

## 项目架构

项目采用模块化架构，主要包含以下核心目录：

```
├── services/          # API服务相关代码
│   ├── config/        # 配置文件
│   ├── modules/       # 业务API模块
│   ├── request/       # Axios封装
│   └── index.ts       # 服务导出
├── stores/            # 状态管理
│   ├── index.ts       # Pinia实例创建
│   ├── setting.ts     # 设置相关状态
│   ├── test.ts        # 测试相关状态
│   └── user.ts        # 用户相关状态
├── types/             # TypeScript类型定义
│   ├── index.ts       # 类型导出
│   ├── music-list.ts  # 音乐列表类型
│   ├── swiper-list.ts # 轮播图类型
│   └── user.ts        # 用户相关类型
├── .env.development   # 开发环境配置
├── .env.production    # 生产环境配置
└── .env.test          # 测试环境配置
```

## 核心功能模块

### 1. API服务模块 (services)

- **统一请求封装**: 基于Axios的LRequest类，提供请求/响应拦截、重试机制、重复请求取消等功能
- **多实例管理**: 支持创建多个请求实例，适应不同API服务需求
- **环境自适应配置**: 根据不同环境自动切换API基础路径
- **业务模块分离**: 将不同业务领域的API请求封装到独立模块

### 2. 状态管理模块 (stores)

- **用户状态**: 管理用户信息、用户歌单等数据
- **设置状态**: 管理界面折叠状态、刷新状态等UI相关状态
- **测试状态**: 提供测试相关的状态管理功能

### 3. 类型定义模块 (types)

- **用户相关类型**: 定义用户信息、歌单、歌曲等核心数据结构
- **业务类型**: 定义音乐列表、轮播图等业务相关数据结构
- **类型导出**: 统一导出所有类型定义，便于在项目中引用

## 环境配置

项目支持三种环境配置，通过.env文件区分：

- **开发环境**: .env.development
- **生产环境**: .env.production
- **测试环境**: .env.test

每种环境可以配置不同的API基础路径、超时时间等参数。

## 项目特点

1. **类型安全**: 全面使用TypeScript，提供完整的类型定义
2. **模块化设计**: 业务逻辑清晰分离，便于维护和扩展
3. **统一的请求处理**: 封装了完整的HTTP请求处理流程，包括错误处理、重试、拦截器等
4. **响应式状态管理**: 使用Pinia管理应用状态，响应式更新UI
5. **环境适配**: 自动适配不同环境的配置需求
